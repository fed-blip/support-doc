
Іноді в issue report можна побачити помилку `Reboot time could not be set`, що означає проблеми в запланованому хард ребуті. Основною причиною цього може бути битий (або відусутність) файлу `crontab`, перевірити чи з ним все в порядку можна через термінал, виконавши команду:
`````bash
cat /etc/crontab
`````
По правильному файл повинен виглядати ось так:
![[Pasted image 20260212094556.png]]
Якщо він виглядає по іншому, це означає що його потрібно замінити, для цього необхідно підключитись до поштомату, зайти в папку де знаходиться файл і замінити його новим ось цим [`crontab`](https://mintinnovations0-my.sharepoint.com/my?id=%2Fpersonal%2Fdzozulia%5Fmint%2Dinnovations%5Fcom%2FDocuments%2Fsupport%20inst%2Fcrontab%20file&viewid=62c09e8d%2D95f8%2D464c%2D9092%2Dda8374827445).
